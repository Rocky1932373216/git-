# git-练习

# 1.先在仓库上创建自己的分支，clone下来在本地，或者命名创建分支 clone

# 2.如果有submodel，clone下来的项目不会clone submodel里的文件，要么切换到
文件clone子仓库，要么命令 git  submodel  init   git  submodel  update

# 3. git pull 是把最新代码和自己本地代码更新并合并

# 4. 如果子仓库代码有修改，提交代码前先切换到子仓库，提交推送后，再切换出来再提交
推送（更新子仓库和主仓库同步）

# 5. git commit -m ""  注释描写情况清楚，干了什么，改了什么，更新了什么。写明白

# 6. 发生冲突时，保留原代码，或者替换原代码

# 7. 多人开发项目时，之间提交代码时，先拉取如master上的最新代码与自己本地分支整合，
然后再提交自己的代码到自己的分支上

# 8. 合并代码时，如有子仓库，确保主仓库和子仓库的代号一样。
